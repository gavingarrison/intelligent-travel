# Product Architecture

## Long-Term Product Shape

The product should be designed as an editorial intelligence platform, not a blog with articles.

The first public surface may look like a publication, but the underlying system should assume that every editorial artifact can become structured knowledge.

The primary site organization is region and location. Verticals such as boba, wine, coffee, hotels, restaurants, family experiences, parks, and museums are lenses across those locations.

## Phases

### Phase 1: Editorial Publication

Core capabilities:

- Original reporting
- Reviews
- Guides
- Region and location coverage
- Vertical coverage, such as boba, wine, coffee, restaurants, hotels, parks, museums, and family experiences
- Curated itineraries
- Author pages
- Editorial taxonomy
- Transparent recommendation rationale

Primary goal: earn trust through a clear point of view.

### Phase 2: Structured Editorial Database

Core capabilities:

- Structured recommendations
- Evaluation fields
- Review metadata
- Experience attributes
- Audience fit
- Seasonality
- Cost context
- Visit context
- Evidence and source tracking

Primary goal: make editorial knowledge reusable instead of trapping it inside prose.

### Phase 3: Editorial Knowledge Graph

Core relationships:

- Region to season
- Location to family profile
- Location to vertical
- Vertical to methodology
- Hotel to service style
- Restaurant to occasion
- Activity to child age
- Experience to memory value
- Place to pace
- Cost to perceived value
- Architecture to atmosphere
- Recommendation to evidence

Primary goal: reveal relationships that improve judgment.

### Phase 4: Contextual Recommendation Engine

Core capabilities:

- Context-aware retrieval
- Preference filtering
- Family-fit matching
- Seasonality matching
- Occasion matching
- Constraint handling
- Explainable recommendations

Primary goal: provide better answers than generic search or review averages.

### Phase 5: Personal Leisure Advisor

Core capabilities:

- Preference memory
- Past trip memory
- Taste calibration
- Family constraints
- Budget sensitivity
- Pace preferences
- Recommendation explanations
- Human editorial override

Primary goal: help people make better personal decisions about meaningful time.

### Phase 6: Leisure Intelligence Platform

Core capabilities:

- Multiple leisure verticals organized across regions and locations
- Membership products
- Premium intelligence products
- Expert reviewer workflows
- Contributor operations
- Proprietary scoring systems
- AI-assisted guidance
- Partner and licensing surfaces

Primary goal: become the trusted intelligence layer for leisure decisions.

## Architectural Implications

The system should support:

- Structured content models
- Editorial workflow states
- Recommendation entities independent from articles
- Evidence-backed scoring
- Human-authored judgment fields
- Machine-assisted retrieval and synthesis
- Long-lived taxonomies
- Versioned methodology
- Source and visit provenance

Avoid assumptions that content is only markdown, pages, or posts.
