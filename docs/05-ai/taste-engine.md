# Taste Engine

The Taste Engine models explicit and inferred preferences.

## Preference Inputs

- Boutique vs Resort
- Quiet vs Lively
- Luxury Threshold
- Design Focus
- Coffee Interest
- Wine Interest
- Children Ages
- Walkability
- Adventure
- Food Priority

## Phase 1 Boundary

Do not build full personalization in Phase 1.

Build the data model and editorial tags so taste can be represented later.

Phase 1 should support taste-aware editorial filters, not private user memory.

## Future Use

- Personalized rankings
- Saved preferences
- AI advisor context
- Better recommendations across locations
